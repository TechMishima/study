## 1.アナログデータとデジタルデータ

### A/D変換の全体像

<b>アナログデータ</b>

時間/値的に連続したデータ<br>
アナログ時計は1秒よりも小さい0.01秒も進みとしては確認できる<br>
<font color="yellow">
音声や映像そのものや、温度や湿度などの情報
</font>

<b>デジタルデータ</b>

とびとびで段階的なデータ<br>
<font color="yellow">
音声や映像を電気信号に変換してコンピューター上で表示
</font>

上記のアナログデータを<br>
①デジタルデータに変換してコンピューター上に保存<br>
②デジタルデータに変換してコンピューターが機器を制御<br>
そのためにA/D変換をしていく

### ビットとバイト

ビット : コンピューターの扱う最小単位で0か1を表現

バイト : ビットを8個集めた単位。256通りの表現ができる。

### 補助単位

大きい/小さい数値を表現する際に使用

<font color="yellow">
下の表はよく使うので暗記してOK！
</font>

| 補助単位 | 意味 |
| :-- | --: |
| k(キロ) | 10の3乗 |
| M(メガ) | 10の6乗 |
| G(ギガ) | 10の9乗 |
| T(テラ) | 10の12乗 |

| 補助単位 | 意味 |
| :-- | --: |
| m(ミリ) | 10の-3乗 |

| n(ナノ) | 10の-9乗 |

「1,000倍」で扱うことが多い

## 2.PCM変換と画像データの変換

アナログデータからデジタルデータへ変換することを「A/D変換」と呼ぶ

### 画像データのA/D変換

ビットマップ方式で表示している<br>
画像をドットの集まりで表示し、<br>
ドットの数、画像の色の種類でデータの大きさが決まる。

- ドットの数

ドットの数が多いほど画像はきれいに表示される。<br>
ドットが多いと「解像度が高くなり」、ドットがすくないと「解像度が低くなる」。

- 画像の色

白黒の場合、2つの情報しかないので、1ビットで完結する。<br>
フルカラーの場合、「24ビット」使うので、16,777,216色存在する。

<b>画像のデータサイズ</b>

画像のデータサイズ = (画像全体のドット数) × (色の情報を保持するビット数)

<font color="yellow">
一般的には8で割ってバイトに変換したうえで、値が大きい時は補助単位を使用している。
</font>
