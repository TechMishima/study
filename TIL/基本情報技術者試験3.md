## 1.アナログデータとデジタルデータ

### A/D変換の全体像

<b>アナログデータ</b>

時間/値的に連続したデータ<br>
アナログ時計は1秒よりも小さい0.01秒も進みとしては確認できる<br>
<font color="yellow">
音声や映像そのものや、温度や湿度などの情報
</font>

<b>デジタルデータ</b>

とびとびで段階的なデータ<br>
<font color="yellow">
音声や映像を電気信号に変換してコンピューター上で表示
</font>

上記のアナログデータを<br>
①デジタルデータに変換してコンピューター上に保存<br>
②デジタルデータに変換してコンピューターが機器を制御<br>
そのためにA/D変換をしていく

### ビットとバイト

ビット : コンピューターの扱う最小単位で0か1を表現

バイト : ビットを8個集めた単位。256通りの表現ができる。

### 補助単位

大きい/小さい数値を表現する際に使用

<font color="yellow">
下の表はよく使うので暗記してOK！
</font>

| 補助単位 | 意味 |
| :-- | --: |
| k(キロ) | 10の3乗 |
| M(メガ) | 10の6乗 |
| G(ギガ) | 10の9乗 |
| T(テラ) | 10の12乗 |

| 補助単位 | 意味 |
| :-- | --: |
| m(ミリ) | 10の-3乗 |
| μ(マイクロ) | 10の-6乗 |
| n(ナノ) | 10の-9乗 |
| p(ピコ) | 10の-12乗 |

※1MBは1,000kB,1,024kBのどちらの表現も正しいが試験では、<br>
「1,000倍」で扱うことが多い

## 2.PCM変換と画像データの変換

アナログデータからデジタルデータへ変換することを「A/D変換」と呼ぶ

### 画像データのA/D変換

ビットマップ方式で表示している<br>
画像をドットの集まりで表示し、<br>
ドットの数、画像の色の種類でデータの大きさが決まる。

- ドットの数

ドットの数が多いほど画像はきれいに表示される。<br>
ドットが多いと「解像度が高くなり」、ドットがすくないと「解像度が低くなる」。

- 画像の色

白黒の場合、2つの情報しかないので、1ビットで完結する。<br>
フルカラーの場合、「24ビット」使うので、16,777,216色存在する。

<b>画像のデータサイズ</b>

画像のデータサイズ = (画像全体のドット数) × (色の情報を保持するビット数)

<font color="yellow">
一般的には8で割ってバイトに変換したうえで、値が大きい時は補助単位を使用している。
</font>

### 音声データのA/D変換（PCM変換）

音のアナログデータをデジタルデータに変換するとき、<br>
「PCM変換」を行う。

- PCM変換
  - 標本化
  - 量子化
  - 符号化

<b>標本化(サンプリング)</b>

一定時間単位でデータを区切り標本を抽出すること<br>
1秒当たりのサンプリング回数を「サンプリング周波数(Hz)」で表す。<br>
1÷サンプリング周波数で表すことができる、サンプリングの間隔の事を「サンプリング周期」と呼ぶ。

<b>量子化</b>

測定したデータを数値化すること<br>
データの段階を決定するビットを「量子化ビット」と呼ぶ。

<b>符号化</b>

量子化で数値化した値を２進数に変換する。

<font color="cyan">
- 標本化でサンプリング周期を短くする(細かく縦に区切る)<br>
- 量子化で量子化ビットを多くする(細かく横に区切る)<br>
上記によって音声データがより正確に変換される(データ量は大きくなる)
</font>

### 過去問演習

<font color="magenta">
色の表現に必要なビット数は2の何乗がを確かめればOK！！<br>
ビットとバイトのどちらなのか注意する！！8倍数値が変わってくる！！
</font>

## 3.機器の制御とD/A変換

### 機器の制御の仕組み

「センサ」が温度や湿度などのアナログデータを取得し「A/D変換」を行う。<br>
デジタルデータを「CPU」に送り、情報処理を行う。<br>
処理したデジタルデータを「アクチュエータ」に送り、電気信号を物理的な操作に変換「D/A変換」を行う。

### シーケンス制御

あらかじめ定められた順序や条件に従って機器を制御

例）洗濯機など

### フィードバック制御

外乱の影響をセンサで検知してコンピューターが修正

例）エアコンなど

### フィードフォワード制御

外乱の影響を予測してあらかじめ修正<br>
※エアコンを例にすると、フィードバックは冷気が入ってきたら対応を行うが<br>
フィードフォワードは冷気が入ってくることを予測して冷気が入ってくる前に対応を行う
