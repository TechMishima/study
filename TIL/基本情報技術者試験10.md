# TIL システム構成要素

## 1.システムの構成

### システム利用形態

- 「バッチ処理」
  - 処理データを一定期間蓄積し、後のタイミングで「一括処理」を行う
- 「リアルタイム処理」
  - 発生「都度処理を」実施する

### システム処理形態

- 「集中処理」
  - 1つのコンピュータが全ての処理を行う
- 「分散処理」
  - 複数のコンピュータで処理を分担

現在は分散処理が主流になっている

### クライアントサーバシステム

- 「クライアント」
  - サービスを要求する
- 「サーバー」
  - サービスを提供する

クライアントが「リクエスト」を送信し、サーバーが「レスポンス」を返す。

クライアントサーバシステムの種類

- 2層クライアントサーバーシステム
  - クライアント側： プレゼンテーション層、アプリケーション層
  - サーバー側 : データベース層
- 3層クライアントサーバーシステム
  - クライアント側 : プレゼンテーション層
  - サーバー側 : アプリケーション層、データベース層

3層クライアントサーバーシステムだと、<br>
クライアントが画面表示のみを担当し、業務処理はサーバーが実施する。<br>
これによりネットワーク通信量抑制とDB処理の効率向上につながる。

### シンクライアントシステム

クライアント端末の機能を必要最低限にとどめ、サーバー側でアプリの実行やデータの管理を行う。<br>
シンクライアントを実現する方法として「VDI」と「DaaS」がある。

- 「VDI」(Virtual Desktop Infrastructure)
  - 自社サーバーに仮想デスクトップを作成
- 「DaaS」(Desktop as a Service)
  - 外部事業者が提供するクラウドに仮想デスクトップを作成

### システムの二重化

- 「デュアルシステム」
  - 同じ処理を「2組のシステムで同時に」行い、結果を照合しながら処理
- 「デュプレックスシステム」
  - 同じ処理に対して2組のシステムを用意し、「1つは障害が発生した時の予備用」
    - 「ホットスタンバイ」 : 起動状態で待機
    - 「コールドスタンバイ」 : 電源オフで待機

### データのバックアップ

- 「フルバックアップ」
  - 対象となるデータを全てバックアップ
- 「差分バックアップ」
  - フルバックアップ以降に変更されたデータをバックアップ
- 「増分バックアップ」
  - 直前のバックアップ以降に変更されたデータをバックアップ

### バスタブ曲線

機械や装置の故障率と時間の関係を表したもの

- 初期故障期
  - 設計ミスや製造エラーなどで故障率が高くなる
- 偶発故障期
  - 初期故障を解消し、故障の発生率が安定
- 摩耗故障期
  - 摩耗や劣化で故障率が高くなる

## 2.システムの信頼性と性能評価

</font>

### システムの性能評価

- スループット
  - 単位時間あたりに処理できる仕事量 「仕事量で評価」
- ターンアラウンドタイム
  - 「データ入力からデータ出力」までにかかる一連の時間 「時間で評価」
  - 主にバッチ処理で使用される指標
    - サーバー側のコンピューター処理に加え、クライアント側のデータ入力とデータ出力の処理時間も加える。
- レスポンスタイム
  - 「コンピューター処理に要する」時間 「時間で評価」
  - 主にリアルタイム処理で使用される指標
    - サーバー側でのコンピューター処理だけの時間

### ベンチマークテスト

- ベンチマークテスト
  - コンピューターの性能を比較・評価するために行うもの
  - 標準的なプログラムを使って同じ条件下でテストを行い、数値化して性能を測定する

### システムの評価特性(RASIS)

- 信頼性(Reliability)
  - システムが安定して稼働し続けられること(MTBFで評価)
- 可用性(Availability)
  - 使いたい時にいつでも使えること(稼働率で評価)
- 保守性(Serviceability)
  - 故障したときに早く復旧できること(MTTRで評価)
- 保全性(Integrity)
  - データが欠損なく正しく保存されていること
- 安全性(Security)
  - 高いセキュリティを維持できること

### MTBFとMTTRの説明

- MTBF 平均故障間隔 (Mean Time Between Failures)
  - システムの故障と故障の間隔の平均時間(システムが正常に稼働している時間の平均)
  - 値が大きいほど故障が起こりにくく「信頼性が高い」
- MTTR 平均修理時間 (Mean Time To Repair)
  - システムが故障した時に修理にかかる平均時間
  - 値が小さいほど修理が早く「保守性が高い」

### MTBFとMTTRの算出方法

例）
```
稼働 : 200時間
修理 : 3時間
稼働 : 100時間
修理 : 1時間
稼働 : 150時間
修理 : 2時間

MTBF = システムが稼働していた時間 / 故障した回数
= 200+100+150 / 3
= 150(時間)

MTTR = 故障中の修理にかかった時間 / 故障した回数
= 3+1+2 / 3
= 2(時間)
```

### 稼働率の説明

「稼働率」 : システムの全運転時間のうち、システムが正常に稼働している時間の割合

```
稼働率 = システムが正常に稼働している時間 / システム全運転時間
= MTBF / (MTBF + MTTR)
```

### 稼働率の算出方法

```
稼働 : 200時間
修理 : 3時間
稼働 : 100時間
修理 : 1時間
稼働 : 150時間
修理 : 2時間

稼働率 = MTBF / (MTBF + MTTR)
= 150 / (150+2)
≒ 0.987(98.7%)
```

### 複数システムの稼働率

複数システムから構成される場合、直列システムか並列システムかによって全体の稼働率が異なる

- 直列システム
  - システム全体が稼働するのは、システムAとシステムB両方が稼働している時
- 並列システム
  - システム全体が稼働するのは、システムAとシステムBの少なくとも一方が稼働している時

```
直列システム
システム稼働率A × システム稼働率B

並列システム
1 - (1-稼働率A)(1-稼働率B)
```

### システムの信頼性設計

システムの信頼性を向上させる設計手法

- フォールトアボイダンス
  - そもそも「故障を起こさない」ように設計
- フォールトトレランス
  - 「問題が起きても大丈夫」なように設計

その他システム信頼性を高める手法

- フェールセーフ
  - 故障した時は「安全を優先させる」
- フェールソフト
  - 故障した時でも機能を低下させ「継続することを優先」
- フールプルーフ
  - 人間が間違った操作をしても、誤作動を起こさないようにする
