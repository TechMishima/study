## 基礎知識

### パソコンの構成要素

- ハードウェア(マウス、キーボードなど)
- ソフトウェア（パワポ、エクセルなど）

### ハードウェアについて

- 五大装置
  - CPU(制御装置と演算装置)
    - 制御装置
    - 演算装置
  - 記憶装置 (主記憶装置、補助記憶装置)
  - 入力装置
  - 出力装置

### ソフトウェアについて

- OS<br>
パソコンを動かすための基本的なソフトウェア
  - パソコン : Windows,macOS,Linux
  - スマートフォン : iOS,Android
- OS以外<br>
ExcelやPowerPointなどのアプリケーション

### Webサービスの仕組み

クライアントとサーバーのやり取り<br>
クライアントからサーバーへ「リクエスト」<br>
サーバーからクライアントへ「レスポンス」<br>
を行うことでWebサービスができるようになっている。

### ブラウザについて

Webサイトを閲覧するために必要なソフトのこと

<b>ブラウザの役割</b>

1. リクエストの送信
2. HTML/CSSを表示する

### サーバーについて

情報の保管、プログラムの実行を行っている大きなコンピュータ<br>
webサーバー、データベースサーバー、メールサーバーなど色々な種類がある<br>
データセンターや社内サーバールームに設置している。

### ドメインとIPアドレス

どのサーバーにリクエストするかを指定している。<br>
ドメインもIPアドレスも表現が違うだけで同じもの。

- ドメイン
  - 例）https://www.~~~~
- IPアドレス
  - 例）208.65.153.238

ドメインだとパソコンでは使用できない。<br>
なぜ？2進数しか処理できないから。

そこで「DNSサーバー」にドメインを送りIPアドレスを取得している。<br>
Domain Name System : ネット上の辞書

### Webサイトとはなにか

- HTML : サイトの構造
- CSS : サイトのデザイン
- JavaScript : サイトの動き

データベース : データの保管庫

データベースからデータを取得するために「サーバーサイドスクリプト」を使用する。<br>
サーバーサイドスクリプトは、php,python,rubyなどがある。

SQL : データの取得/追加/更新/削除など、データベースを操作する言語

データベースとのやり取りの部分を「サーバーサイド」と呼ばれる。<br>
パソコンに表示されるやり取りの部分を「フロントエンド」と呼ばれる。

### 基本情報技術者試験とは

会社が新しいシステムを構築する際に必要な知識を網羅的にまとめた試験


### どんなことを学習していくのか

- 正しい戦略
  - システムの活用法やシステム化の妥当性を検討
  - 他社を分析する為の財務・マーケティング知識
  - その他ITに関する法務知識
    - ストラテジ分野で学習
- システム構築スキル
  - システムの構築
  - システム稼働の保証や障害発生時の対応
  - システム構築作業のマネジメント
    - マネジメント分野、システム構築要素、システム開発で学習
- システム構築要素の理解
  - プログラミング言語、データベース、ハードウェア、ソフトウェア等システムを構築する要素の理解
  - ネットワークや情報セキュリティの理解
    - アルゴリズムとプログラミング、データベース、情報セキュリティ、ハードウェア、ネットワーク、ソフトウェアで学習
- コンピュータ理論の理解
  - コンピュータが行っている処理の原理を理解
    - 基礎理論およびA/D変換で学習

### 分野ごとの出題数

- ストラテジ分野 : 12問
- マネジメント分野 : 7問
- それ以外（テクノロジ分野） : 41問

### 学習のコツについて

1. 基礎理論及びA/D変換<br>
  過去問を解きまくる。練習が必要な知識が多いため
2. ハードウェア<br>
  重要なポイントを中心に、インプットをしっかりと行う
3. ソフトウェア<br>
  カタカナが多いが難しくはない。用語の暗記ではなく、意味の理解を優先する
4. データベース<br>
  暗記する範囲と頭を使う範囲がある。集中できる環境があると良い
5. ネットワーク<br>
  暗記する範囲と頭を使う範囲がある。<br>
  馴染みがないため難しく感じる。腹落ちしなくても出題ポイントが決まっているため過去問演習で乗り切れる
6. 情報セキュリティ<br>
  B試験でも出題されるため、最も重要な分野。<br>
  背景を理解しながら暗記をしていく。<br>
  しかしサイバー攻撃の手法は丸暗記の為、聞き流しの学習でＯＫ
7. アルゴリズムをプログラミング<br>
  B試験アルゴリズム問題に関係する。<br>
  フローチャートとオブジェクト指向は腹落ちするまで学習する。<br>
  その他の分野は過去問演習で学習してもOK
8. システム構成要素<br>
  システムの稼働率はほぼ毎回出題されているため、必ず計算できるようにしておく
9. システム開発<br>
  勉強している内容が、システム開発分野のどこに位置するのか意識しながら学習する
10. マネジメント分野<br>
  幅広く出題されるが、癖の強い問題は出題されにくい
11. ストラテジ分野<br>
  アルファベットで似たような単語が多く登場する<br>
  繰り返し倍速で学習し、大まかと理解し、過去問演習で用語を定着させていく
