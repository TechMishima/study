# TIL ソフトウェア

## 1.基本ソフトウェア(OS)

### ソフトウェアの概要

- 基本ソフトウェア
  - Operating System(OS)の略でコンピューターの基本的な動作を担う。
    - その中で中核を担うソフトを「カーネル」と呼ぶ。
- ミドルウェア
  - 多くのアプリケーションが共通して必要とするが、OSでは提供できないような機能を持つソフトウェア
- 応用ソフトウェア
  - 普段使っているアプリケーション

### オペレーティングシステム

OS : どのコンピューターでも普遍的に必要とされる機能を担うソフトウェア

### GUIとAPI

- GUI(Graphical User Interface)
  - 視覚的に操作して命令を伝える画面
- API(Application Programming Interface)
  - アプリケーションからOSが保持している各種機能を利用するために仕組み

### ソフトウェアコースの概要

1. ジョブ/タスク管理<br>
  CPUをどのように使用するか管理すること
2. 記憶管理<br>
  CPUの「プログラム内蔵方式」による記憶領域の管理
3. ファイル管理<br>
  ファイルやフォルダの管理もOSが行っている

## 2.ジョブ管理とタスク管理

<font color="magenta">
試験問題頻出で重要な項目！！
</font>

### ジョブとタスク

- ジョブ : 人間からみた仕事の単位
- タスク : コンピューターからみた仕事の単位、ジョブを細かく分割したもの

「ジョブ管理」「タスク管理」 : どのジョブ/タスクをいつCPUに作業させるかOSが管理している

### ジョブ管理

ジョブを与えられたコンピューターはジョブを「入力待ち行列」に入れる。<br>
随時、タスク処理の方へ送る。<br>
タスク処理されたジョブは、「出力待ち行列」に入れる。<br>
プリンタなどは、出力待ち行列のジョブを実行している。

CPUは処理を行ったジョブを記憶媒体の出力待ち行列に入れることにより、他の処理を行えるようになっている。<br>
この動作を★「スプーリング」と呼び、単位時間あたりに処理できる仕事量(「スループット」)が向上する。

### タスクの状態遷移

3つの状態を遷移しながら処理している
1. 実行可能状態 : CPUの処理を待っている状態
2. 実行状態 : タスクが処理されている状態
3. 待機状態 : 入出力処理の終了を待っている状態

- ディスパッチャ : CPUの割当を実行するプログラム
- ディスパッチ : CPUを割当てること

### スケジューリング

- スケジューリング : ディスパッチするルールの事

1. 到着順方式(ノンプリエンプティブ)<br>
  実行可能状態になった順番にCPUの使用権を割り当てる
2. 優先順方式(プリエンプティブ)<br>
  優先順位が高いタスクから順にCPUの使用権を割り当てる
3. ラウンドロビン方式(プリエンプティブ)<br>
  一定時間ごとにCPU使用権を切り替える

- プリエンプティブ方式
  - OSがCPUを管理し、実行中のタスクを途中で中断して他のタスクにCPU使用権を割当てる
- ノンプリエンプティブ方式
  - 実行中のタスクを途中で中断せずプログラムに処理を任せる

### マルチプログラミング・マルチタスク

<font color="magenta">
マルチタスクの計算問題は頻出！！
</font>

入出力処理を行って待機状態の場合、他のタスクを行い、有効活用したい。

- マルチプログラミング・マルチタスク
  - 待機状態の時、他のタスクに割り当てCPUを有効活用すること

### 割込み処理

<font color="magenta">
割り込み処理の問題も頻出！！
</font>

<b>内部割込み(プログラムが要因)</b>
- プログラム割込み
  - 処理することが出来ない不正な命令により発生(0での割り算やオーバーフローなど)
- SVC割込み
  - プログラムがOSに入出力要求したとき等に発生

<b>外部割込み(プログラム外の要因)</b>
- 機械チェック割込み
  - 電源の異常や装置の故障など、ハードウェアの不具合で発生
- タイマ割込み
  - 設定された時間が経過してプログラムが中断されることで発生
- コンソール割込み
  - PCを操作している利用者の介入が行われた際に発生
- 入出力割込み
  - 入出力装置の動作完了時や中断時に発生

## 3.実記憶管理

プログラム内蔵方式により、補助記憶装置から主記憶装置へプログラムを読み込む際、主記憶装置の容量をOSが管理している。<br>
その読み込みのルールのことを「プログラム読み込み方式」と呼ぶ。

### 固定区画方式

主記憶装置の容量をあらかじめ固定で分けておく。<br>
しかし8MBの領域に4MBを入れると無駄領域が発生し、12MBのプログラムは読み込むことが出来ない。<br>
処理は高速だが、処理効率は悪い。

### 可変区画方式

読み込む際に適切な大きさに区画を区切る方式。<br>
無駄な領域がなく、大容量プログラムの読み込みが可能になっている。

プログラムが完了されると、随時消されていくが、区画がバラバラの状態で保持されてしまう。<br>
この区画がバラバラで保持している状態を「フラグメンテーション」と呼ぶ。<br>
その時、バラバラの区画をまとめる作業を「メモリコンパクション」と呼ぶ。

<font color="yellow">
主記憶装置をぎゅっとする : 「メモリコンパクション」<br>
ハードディスクをぎゅっとする : 「デフラグ」
</font>

### オーバーレイ方式

主記憶装置の容量以上のプログラムを実行する際に使用する。<br>
1つのプログラムを「セグメント」と呼ばれる独立した処理単位に分割し、必要な部分だけを使用する。

### スワッピング方式

優先度の高いタスクを行うときに、「スワップイン」と「スワップアウト」を行う方式。<br>

- スワップアウト
  - 中断したプログラムを主記憶装置から補助記憶装置に一時退避
- スワップイン
  - 処理を再実行する際に主記憶装置に呼び戻す

### メモリリーク

「メモリリーク」 : OSのバグ等により処理をしていないのに使用不可領域が発生すること

「ガーベージコレクション」 : メモリリークの領域を解放すること

## 4.仮想記憶管理

- 実記憶
  - 物理的な記憶領域
  - 容量の制限あり
- 仮想記憶
  - プログラムの整理不要
  - 好きなだけ読み込み可能

### 仮想記憶の仕組み

主記憶装置の他に補助記憶装置を用いて、見かけ上プログラムを仮想記憶領域という空間に読み込ませている

### 仮想アドレスと動的アドレス変換

見かけ上、プログラムに1番、2番と「仮想アドレス」が割り振れていて、
それにより主記憶装置か補助記憶装置か判断している。<br>
CPUの「メモリ変換ユニット」が紐づけ作業を実施している。<br>
上記のような仕組みを「動的アドレス変換機構」もしくは「DAT」(Dynamic Address Translation)と呼ぶ。

### ページング方式

- ページング方式
  - 装置プログラムや仮想記憶領域を「ページ」と呼ばれる細かい単位に分割して管理する

すぐに必要な命令は主記憶装置に、一旦は不要な命令は補助記憶装置に管理する

### ページフォールト

仮想記憶領域にプログラムが読み込まれたが、CPUが主記憶装置に欲しいプログラムが無かった時にエラーになることを「ページフォールト」と呼ぶ。

### ページインとページアウト

- 「ページイン」
  - ページフォールトが起こったら、補助記憶装置から必要なページを主記憶装置に持ってくる
- 「ページアウト」
  - ページイン時に主記憶装置の容量が一杯である場合、補助記憶装置に不要なページを退避

「スラッシング」 : ページイン/ページアウトが多発すると処理効率が低下する

<font color="cyan">
広い意味ではスワッピングとページイン/アウトは同じ処理を行っている
</font>

### ページング置き換えアルゴリズム

適当なページを退避させると、すぐにページインが発生する懸念があるため、退避させるページはよく検討する。

- FIFO(First In First Out)
  - 一番最初にページインしたページを追い出す
- LIFO(Last In First Out)
  - 一番最後にページインしたページを追い出す
- LRU(Least Recently Used)
  - 最も長い時間参照されていないページを追い出す
- LFU(Least Frequently Used)
  - 最も参照回数の少ないページを追い出す

## 5.ファイル管理

ディレクトリ : 複数のファイルやディレクトリをまとめて管理するもの(=フォルダ)

ディレクトリは階層構造で構成されている。

### ディレクトリ名称

- カレントディレクトリ : 今いるディレクトリ
- 親ディレクトリ : カレントディレクトリの1個上
- サブディレクトリ : ディレクトリの中にあるディレクトリ
- ルートディレクトリ : 階層構造の一番上に位置するディレクトリ
- ホームディレクトリ : ログイン直後に位置するディレクトリ

### ファイルパス

ファイルパス : 目的ファイルが保管されているディレクトリの経路

- 絶対パス : ルートディレクトリから目的ファイルまでの経路
- 相対パス : カレントディレクトリから目的ファイルまでの経路

<b>[ファイルパス記載方法]</b>
1. ルートディレクトリは省略されると考える
2. ディレクトリとディレクトリの間は「￥」もしくは「/」で表記
3. カレントディレクトリは「.」で表記
4. 1階層上のディレクトリ(親ディレクトリ)は「..」で表記

※親ディレクトリへのつなぎ方
```
./../
カレントディレクトリの親ディレクトリ
```

## 6.マルチメディア

### 圧縮と解凍

容量が大きいファイルを小さくさせる作業を「圧縮」と呼ぶ。<br>
逆に圧縮されたデータを元に戻す作業を「解凍」と呼ぶ。

- 可逆圧縮方式 : 圧縮データを完全に復元できる
- 不可逆圧縮方式 : 圧縮データを完全に復元できない

### 代表的なファイル形式

- 静止画
  - BMP : 画像が圧縮されないため容量が大きい
  - GIF : 256色の可逆圧縮
  - PNG : 256色・もしくはフルカラーの可逆圧縮
  - JPEG : フルカラーの「不可逆圧縮」。「国際標準規格」
- 動画
  - MPEG : 「不可逆圧縮」。「国際標準規格」

### 画像処理技術(Computer Graphics)

たまにでる程度、頭の片隅に置いておくぐらいの認識でOK

- アンチエイリアシング
  - 画像内に発生するギザギザなど、傾いた直線を滑らかにする技術。(特定の場所をぼかす)
- テクスチャマッピング
  - 画像に柄や模様などを貼り付け、質感や立体感を表現する技術
- モーフィング
  - ある物体画像変化の中間をコンピューターで演算し、変化の過程を表現する技術
- レイトレーシング
  - 画像内の水たまりなどの光の反射をよりリアルに表現する技術
- クリッピング
  - ある領域からはみ出した画像を切り捨てる作業
- シェーディング
  - 物体画像に影を入れて、より立体感を出す技術

### VRとAR

- VR(Virtual Reality)
  - 仮想現実。「仮想の空間」に入り込む感覚を与える技術
- AR(Augmented Reality)
  - 拡張現実。「現実世界の中に」仮想の情報を与える技術
  - ポケモンGOなど
